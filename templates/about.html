About This Platform

RimAI

An AI-powered agricultural intelligence platform for Zimbabwe, built for the AI4I Challenge 2026 Development Track. "Rima" means "to plant/cultivate" in Shona.

Vision

RimAI is the intelligence layer connecting Zimbabwe's farmers, AGRITEX extension officers, and policymakers through predictive, explainable agricultural AI — not a single-purpose farming app, but a shared foundation for data-driven agricultural decision-making at every level, from an individual farm to national food-security planning.

What the MVP Does
  • Farmer Dashboard: Farm Health Score, AI Daily Brief, yield and risk prediction, an economic impact simulator, and a Virtual Agronomist that answers questions using the farmer's own analysis.
  • AGRITEX Dashboard: A farmer priority queue and ward-level risk table ranked by urgency, so one officer can see which farms need a visit most, with the specific cause (rainfall deficit, pest pressure, rotation depletion) attached to each — not just a bare risk badge.
  • Ministry Dashboard: A national food security index and an interactive Zimbabwe province map (built from real administrative boundary data) with province-level risk, yield, and the primary driver behind each province's risk level.
  • Admin Dashboard: Live model performance metrics, data pipeline health, and a Data Provenance & Transparency page disclosing exactly which datasets are real, synthetic, or hybrid.
  • Explainable predictions, everywhere: Every risk score is paired with cited, factor-level reasoning (rainfall anomaly, crop rotation history, pest pressure) rather than a bare number.
Technology Stack
FrontendHTML, Bootstrap 5, JavaScript, Plotly
BackendPython Flask
Yield PredictionRidge regression (walk-forward CV R\u00b2 = 0.824)
Risk ClassificationXGBoost (CV accuracy 78.8%, macro F1 0.735)
Virtual AgronomistRule-grounded explanation engine — deliberately not a hosted LLM, for offline reliability, zero marginal cost, and no hallucination risk
Weather DataNASA POWER API (real, satellite-derived), with a disclosed fallback dataset if unreachable
Yield HistoryFAOSTAT (real, 2000\u20132024), with a disclosed fallback if unreachable
DatabaseSQLite (MVP) \u2192 PostgreSQL (production)
AuthFlask sessions + Werkzeug password hashing, role-based access control
Roadmap

Phase 2: live WhatsApp alerting, an LLM layer added alongside (not replacing) the deterministic explanation engine for free-text queries, image-based crop/pest diagnosis, and an AGRITEX pilot in 1\u20132 districts.

Phase 3: government data-sharing integration, satellite-derived vegetation indices replacing the current NDVI proxy, optional IoT soil/weather sensors, and continuous model retraining as real data volume grows.

Every synthetic dataset used in this MVP is disclosed on the Data Provenance & Transparency page and is designed to be automatically displaced by real data as it becomes available, without a dashboard rebuild.
